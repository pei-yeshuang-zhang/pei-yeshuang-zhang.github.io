JavaScript and its relationship to HTML and CSS

Imagine a construction process. The HTML is the engineer, CSS is the interior designer, and JavaScript is the project manager.

When you build the basic structure using HTML, it's like an engineer is preparing an architectural blueprint; they need to figure out the skeleton of the building, for example, how many floors are there in this building and what is each floor used for.

Once the HTML has completed the web layout, the next step is to make the website pages look better, just as interior designers need to choose paint colors and flooring materials for a building.

Then comes JavaScript; its job is to give the web page functionality. Just like the project manager needs to ensure that every part in the building is functioning properly and coordinate with engineers and interior designers.

Control flow and loops

Control flow in JavaScript is how your computer runs code from top to bottom. It starts from the first line and ends at the last line unless it hits any statement that changes the program's control flow, such as loops. When it hits a loop, that loop will pause the control flow and repeat a block of code until a specific condition is met.

What is the DOM

DOM stands for Document Object Model. In the world of web, all HTML webpages are referred to as documents. The Document Object Model represents each of these web pages in a tree-like structure to make it easier to access and manage the elements.

In JavaScript, DOM is actually a class that makes it really easy to define each element present within a web page so it can be modified later. Whenever a page is loaded in the browser, its document object is formed that is named as “document”. You can apply built-in methods on this object to access and modify the HTML elements.

What functions are and why they are helpful

Simply put, a function allows you to define a block of code, give it a name and then execute it as many times as you want. This eliminates the need of writing the same code again and again. It helps programmers in writing modular codes. Functions allow a programmer to divide a big program into a number of small and manageable functions.

Imagine you are running a small business, and have over a hundred customers. Every time a new customer signs up on your website, you have to send them a greeting email. Instead of writing the same email every time, we could just simply write a function and call it (run it) when you need to send a new customer a repetitive email.
